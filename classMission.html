Detailed Description

template<typename T>
class Mission< T >

This is the mother class of the other mission classes. It is a template, which means that single or double precisions may be used to represent real numbers. Its main objective is the definition of the interface among all mission classes, as well as the definition of the common attributes and methods among them, such as the 3D position (x, y, h), the heading and the gamma angles. The point (x, y, h) represents the place where each mission is executed. The squadron must arrive at (x, y, h) by assuming the given heading and gamma angles, which are respectively the squadron's angles in turn of the vertical axis and the latitudinal axis.

template<typename T >
virtual Mission* Mission< T >::get_starting ( )
pure virtual

This is an abstract method, i.e., it does not have a body in this class, but the children classes of the class Mission have to define each one a body for it. The importance of this method is the definition of an interface, since the same method in all children classes must have the same signature, which open the door for the implementation of generic algorithms.

Returns
All methods that implement this abstract one must return an object of the class Mission, which may be MissionLongitudinal, MissionLatitudinal or MissionCircular.

◆ run_mission()

template<typename T >
virtual vector<vector<T> > Mission< T >::run_mission ( Mission< T > *  anterior,
MissionSolver< T > *  ms,
vector< vector< T > > &  ref 
)
pure virtual

This is the most important method of this abstract class, because it defines the interface of the method run_mission, to which every child class must obey and implement. This method (when implemented) is responsible for producing references that take the squadron (and consequently each aircraft) to execute the specified mission. Since the radius and angles of the circular maneuvers and length of the straight forward maneuvers are all known, each aerial vehicle's trajectory can therefore be deterministically calculated for all missions.

Parameters
anteriorA pointer to an object of the class Mission, which was executed immediately before this one.
msA pointer to an object of the class MissionSolver, which has several attributes that are used to execute the specified mission.
refThe vector of references that are populated by this method, i.e., this attribute is passed by reference, which means that it is altered by this method. This vector of references is used to make the squadron execute the specified mission.
Returns
The same value of the parameter ref.

Implemented in MissionLongitudinal< T >, MissionLatitudinal< T >, and MissionCircular< T >.
