Detailed Description

template<typename T>
class MissionSolver< T >

This class is responsible for implementing the unification framework, i.e., it implements the integration among the other classes. Besides, it concentrates the most part of the source-code. According do these purposes, this class stores important attributes, such as the number of aerial vehicles in the squadron, the index of the aircraft, the delta distance, the radius for latitudinal formations, the initial and standard velocity and the number of changes, i.e., the number of times the squadron has re-configured between longitudinal and parallel formations or vice-versa. Besides, this class stores the vector of missions and of starting points, where the last represents the previous one translated to the place where it has to begin so that the aircraft executes the specified mission accordingly.

◆ insert_mission()

template<typename T >
void MissionSolver< T >::insert_mission ( Mission< T > *  m)

This method inserts the specified mission, which may be longitudinal, latitudinal or circular, in the vector of missions. Although the data structure used to store the missions is a vector, the missions are executed in a first-in first-out (FIFO) fashion. The vector is used due to simplicity reasons. This method does not return any information.

Parameters
mThe pointer to an object of the class mission, which is polymorphic.

◆ proximidade_paralela()

template<typename T >
proximidade_t MissionSolver< T >::proximidade_paralela ( Mission< T > *  origem,
Mission< T > *  destino 
)
private

This method says if, after turning an angle of PI/2 rad, the next starting point is near (PERTO) or far (LONGE). The decision is taken based on the value of |t0|, which is the x coordinate of the crossing point (t0, t1). If |t0| >= 2 x d + 4 x r, then it is considered far, where d is the length of the squadron in longitudinal formation and r is the radius of the movements performed during latitudinal re-configurations. Otherwise, it is considered near.

Parameters
origemThe current starting mission.
destinoThe next starting mission.
Returns
PERTO (near) or LONGE (far), with regard to the destination.

This method says if, after turning an angle of PI/2 rad, the next starting point is near (PERTO) or far (LONGE). The decision is taken based on the value of |t1|, which is the y coordinate of the crossing point (t0, t1) between the current and the next positions. If |t1| >= 2 x d + 3 x r, then it is considered far, where d is the length of the squadron in longitudinal formation and r is the radius of the movements performed during latitudinal re-configurations. Otherwise, it is considered near.

destinoThe next starting mission.

◆ run_missions()

template<typename T >
vector< vector< T > > MissionSolver< T >::run_missions

This method determines the vector of references (3D points) that command each aerial vehicle to execute the multi-functional missions queue accordingly. Since the method "run_mission" is known for each mission, this method only calculates the starting points of each mission (by calling the method get_starting) and call the method "run_mission" for each element of the vector, which is processed in a FIFO order (in a "for" loop).
